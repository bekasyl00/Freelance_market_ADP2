2. System Architecture

The system is built on a modern microservices architecture, ensuring high availability, fault tolerance, and independent scalability of components. The backend comprises three core Go-based microservices, an API Gateway, and an event-driven Notification Service.

3. Microservices Implementation Details

Each microservice is developed using Clean Architecture. Dependencies flow inwards: from Delivery (Handlers) to Usecases (Business Logic) to Repositories (Data Access). This pattern prevents tight coupling and makes the system easily maintainable.

3.1 User Service

Handles authentication, role-based access control (Admin, Client, Freelancer), and profile management. It securely hashes passwords using bcrypt and issues JWT tokens for stateless session management across the cluster.

3.2 Job Service

Manages the lifecycle of freelance gigs (Open, In Progress, Completed). It implements an intelligent caching mechanism to reduce database load when retrieving the list of open jobs. Upon a job creation or an application submission, it publishes an event payload to the Message Queue.

3.3 Payment Service

Responsible for financial transactions and wallet balances. The service utilizes complex PostgreSQL transactions to ensure strict ACID compliance. Escrow logic is implemented to securely hold funds during an active job and release them only upon successful completion.

4. Inter-Service Communication (Message Queue)

To prevent synchronous API blocking, the system utilizes NATS as a Message Broker. This fulfills the strict requirement for asynchronous message queues.

  • Publisher: When a significant action occurs (e.g., a freelancer applies to a job), the Job Service publishes a JSON payload to a specific NATS topic.
  • Subscriber: The Notification Service continuously listens to these topics. Upon receiving a message, it independently formats and sends an SMTP email to the relevant user without slowing down the primary API response.

5. Frontend & Cloud Deployment

The client-facing application is a highly responsive Single Page Application (SPA) built with Vue.js (Vite). It provides dynamically rendered, role-based interfaces for Clients and Freelancers.

The frontend is configured for Continuous Deployment (CD) using Vercel, ensuring that any changes to the user interface are automatically built and published globally via edge networks.

6. Observability and Monitoring (SRE)

An enterprise-grade observability stack guarantees that the system's health can be monitored in real-time, completely fulfilling the final bonus requirement.

  • Prometheus: Scrapes application metrics (request rates, latency, error counts) from all microservices.
  • Loki & Tempo: Provides centralized log aggregation and distributed tracing to track the lifespan of requests across the microservice network.
  • Grafana: Visualizes the aggregated data through interactive dashboards and manages alerting rules.
